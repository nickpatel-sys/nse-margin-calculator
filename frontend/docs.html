NSEMargins

A local web application for calculating NSE F&O SPAN margins on a portfolio of futures and options positions. It downloads official end-of-day data from NSE archives, stores it in a local SQLite database, and provides a REST API consumed by a browser-based portfolio calculator.

Quick Start

# Activate virtualenv (Windows)
venv\Scripts\activate

# Start development server
python run.py

# Open in browser
# http://localhost:5000

Data is downloaded automatically on startup for the most recent trading day. The scheduler re-fetches every evening at 18:30, 19:00, and 19:30 IST (Mon–Fri).

Data Modes

The app operates in one of two modes depending on which NSE files are available:

ModeSourceAccuracy
Live SPAN SPAN XML (nsccl.{YYYYMMDD}.s.zip) + bhavcopy Exact — uses NSE's official 16-scenario risk arrays
Estimated Bhavcopy only Approximate — PSR estimated from config rates; typically within 2–5% of exchange values

The status chip in the top-right corner of the UI shows the current mode.

Adding Positions

Options

  1. Select a Symbol (e.g., NIFTY, BANKNIFTY, RELIANCE).
  2. Select Option type (this is the default).
  3. Pick an Expiry date from the dropdown.
  4. Pick a Strike — all strikes available for that expiry are listed.
  5. Choose CE (call) or PE (put).
  6. Set Side: Buy (long) or Sell (short). Default is Sell.
  7. Set Lots (1–5000).
  8. Click Add to Portfolio or press Enter.

Futures

  1. Select a Symbol.
  2. Select Future type.
  3. Pick an Expiry.
  4. Set Side and Lots.
  5. Click Add to Portfolio.
💡

Adding the same contract key + side combination again merges into the existing row — it adds to the lot count rather than creating a duplicate.

Each row shows the contract name, lot count, side, expiry, and estimated notional value.

  • Prev. Close (futures only) — enter yesterday's settlement price to calculate variation margin. Leave blank to omit.
  • ✕ Remove — removes a single position.
  • Clear All — empties the entire portfolio.

Variation Margin (MTM)

Variation margin is the daily mark-to-market cash flow on futures that NSE collects or credits each evening.

  • Gain (price moved in your favour): positive number — reduces Net Cash Required.
  • Loss (price moved against you): negative number — increases Net Cash Required.
📐

Example: Short 1 lot NIFTY future. Yesterday's close ₹22,400, today's settlement ₹22,200.

VM = 1 lot × 75 (lot size) × (22,200 − 22,400) = −₹15,000 (loss — you must pay this).

Net Cash Required = Total Margin + ₹15,000.

💡

Mixed portfolios (long + short options on the same underlying) benefit from SPAN netting — the worst combined scenario is less than the sum of individual worsts.

Long options pay no exposure margin — the premium is the maximum loss.

If results show Estimated mode, refresh after 18:30 IST to get official SPAN data.

Margin Engine

NSE uses the SPAN (Standard Portfolio Analysis of Risk) methodology. It evaluates a portfolio's loss across 16 hypothetical market scenarios and charges the worst-case outcome, plus a separate exposure margin on notional value. The engine lives in backend/margin/calculator.py.

Calculation Steps

1. Load Contracts

Each position carries a contract_key. The engine fetches the matching Contract and RiskArray rows from the database for the given trade date.

2. Group by Commodity

All positions on the same underlying (e.g., all NIFTY contracts regardless of expiry or strike) are grouped into a commodity group. SPAN netting happens within each group.

3. Per-Commodity SPAN

a. Scenario P&L

For each of the 16 SPAN scenarios, the net portfolio P&L is the sum across all positions:

scenario_pnl[s] = Σ (quantity_in_lots × risk_array[s])

risk_array[s] is already scaled per lot. Positive = loss for a long position.

b. Worst-case scan risk

Scenarios 15–16 are extreme moves (±2 × PSR). Only 35% of the extreme-scenario loss is charged:

adjusted = [pnl if s < 14 else pnl * 0.35 for s, pnl in enumerate(scenario_pnl)]
scan_risk = max(adjusted)   # highest loss across all 16

c. Short option minimum (SOMC)

somc = |quantity| × lot_size × short_option_min_charge

d. Commodity SPAN

commodity_span = max(scan_risk, short_option_minimum)
📌

All-long-option commodity: if every position is a long option, scan_risk is zeroed — the premium already paid caps the maximum loss.

4. Intra-Commodity Spread Charges

Calendar spread charges apply when a commodity has positions in multiple expiries with opposite signs (e.g., long near-month + short far-month future). A simplified charge is applied to matched spread units.

5. Inter-Commodity Spread Credits

After per-commodity SPAN, the engine looks for delta-matched hedges across commodities. These reduce total SPAN.

Spread Pairs

Leg 1Leg 2CreditDelta Ratio
BANKNIFTYNIFTY50%1 : 3
FINNIFTYNIFTY50%1 : 2
MIDCPNIFTYNIFTY50%1 : 2

Matching: compute net delta per commodity → check for opposite-signed deltas → credit = spread_units × credit_rate × min(scan_risk_leg1, scan_risk_leg2).

Exposure Margin

Instrument TypeRateNotes
FUTIDX, OPTIDX2%Per NSE circular NSCCL/CMPT/46514
FUTSTK, OPTSTK5%
Long options0%Premium is the maximum loss
 
notional = |quantity| × lot_size × underlying_price
exposure = notional × rate

The 16 SPAN Scenarios

The risk_arrays table stores 16 values (s01–s16) per lot for each contract.

ScenarioPrice MoveVol Move
10Up
20Down
3+⅓ PSRUp
4+⅓ PSRDown
5−⅓ PSRUp
6−⅓ PSRDown
7+⅔ PSRUp
8+⅔ PSRDown
9−⅔ PSRUp
10−⅔ PSRDown
11+1 PSRUp
12+1 PSRDown
13−1 PSRUp
14−1 PSRDown

16−2 PSRExtreme — 35% cover applied

The SPAN XML contains per-unit values. The parser multiplies by lot_size before storing, so the calculator uses quantity_lots × risk_array[s] with no further scaling.

Estimated Mode (Fallback)

When the SPAN XML is unavailable, risk arrays are synthesised in fallback_rates.py:

psr_inr = fallback_psr_rate × underlying_price
SymbolRate
NIFTY, SENSEX8%
BANKNIFTY, FINNIFTY, MIDCPNIFTY9%
All stocks15%

Futures use delta = 1.0. Options use a delta approximation: ATM ≈ 0.5, interpolated from moneyness. Vega and time decay are not modelled in estimated mode.

Data Pipeline

NSEMargins fetches data from two NSE archive endpoints. The pipeline runs automatically on startup and on a nightly schedule; it can also be triggered manually via the API.

Data Sources

1. UDiFF Bhavcopy REQUIRED

https://nsearchives.nseindia.com/content/fo/
  BhavCopy_NSE_FO_0_0_0_{YYYYMMDD}_F_0000.csv.zip

Provides: all F&O contracts, lot sizes, settlement prices, previous day close, underlying prices, instrument types. Cached to data/fo_bhavcopy_{YYYYMMDD}.csv.zip.

If the bhavcopy download fails, the pipeline aborts for that date.

2. SPAN XML SUPPLEMENTARY

https://nsearchives.nseindia.com/archives/nsccl/span/
  nsccl.{YYYYMMDD}.s.zip

Provides: official 16-scenario risk arrays, official PSR/VSR per underlying, composite delta. Cached to data/span_{YYYYMMDD}.zip.

If this download fails the pipeline continues in Estimated mode — margins use fallback PSR rates.

3. ISIN Map ONE-TIME

NSE EQUITY_L.csv, downloaded once and cached in data/equity_isin.json. Provides ISIN codes for stock underlyings. Index instruments receive null.

Processing Flow

download_for_date() ├─ Fetch bhavcopy zip ───────► data/fo_bhavcopy_{date}.csv.zip └─ Fetch SPAN XML zip ───────► data/span_{date}.zip (if available) ↓ orchestrator.parse_downloaded_file() ├─ bhavcopy_parser.parse_bhavcopy() │ └─ Creates Contract rows + CombinedCommodity (estimated PSR) └─ span_xml_parser.parse_span_xml() (only if span zip exists) └─ Creates RiskArray rows └─ Updates CombinedCommodity with official PSR/VSR └─ Sets is_estimated = False

Bhavcopy always runs first — the SPAN XML parser relies on Contract rows already being present to match risk arrays.

Scheduler

Three APScheduler cron jobs run Monday–Friday IST:

Time (IST)Behaviour
18:30Primary — forces refresh even if data already present
19:00Retry — skips if already succeeded
19:30Final retry — skips if already succeeded

NSE typically publishes bhavcopy by 18:00–18:15 and the SPAN XML by 18:30–18:45. The retries ensure the SPAN XML is captured even when delayed.

Caching & Skip Logic

ConditionBehaviour
SpanFile.parse_status == 'success'Startup + scheduler skip silently
Bhavcopy zip on diskRe-uses cache; no re-download
SPAN XML zip on diskRe-uses cache; no re-download
force=True (manual refresh)Deletes existing DB data for the date, re-parses from cache (or re-downloads if zip missing)

Cached zip files in data/ are never automatically deleted — they serve as a replay buffer if re-parsing is needed after code changes.

Deployment — EC2 Free Tier

NSEMargins ships with a Dockerfile and docker-compose.yml. The steps below deploy it on an AWS EC2 t2.micro instance — within the free tier.

ℹ️

Why single worker? The app uses SQLite (single writer) and APScheduler (must run in one process). Gunicorn is configured with --workers 1 --threads 4 — one process, four concurrent request threads.

Step 1 — Launch an EC2 Instance

  1. Open the EC2 Console → Launch Instance. Name it nsemargins.
  2. AMI: Ubuntu Server 24.04 LTS (free tier eligible).
  3. Instance type: t2.micro (free tier).
  4. Key pair: select or create one — save the .pem file.
  5. Security group: allow SSH (port 22) from your IP; HTTP (port 80) from anywhere.
  6. Storage: 20 GB gp3. Launch and note the Public IPv4 address.

Step 2 — Install Docker on the Instance

ssh -i your-key.pem ubuntu@<EC2_PUBLIC_IP>

# Install Docker
sudo apt-get update
sudo apt-get install -y ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg \
  -o /etc/apt/keyrings/docker.asc
echo "deb [arch=$(dpkg --print-architecture) \
  signed-by=/etc/apt/keyrings/docker.asc] \
  https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io \
  docker-buildx-plugin docker-compose-plugin

# Allow running docker without sudo
sudo usermod -aG docker ubuntu && newgrp docker

Step 3 — Copy the App to EC2

From your local machine:

rsync -avz --exclude='venv/' --exclude='data/' --exclude='.git/' \
  -e "ssh -i your-key.pem" \
  "path/to/NSE margin calculator/" \
  ubuntu@<EC2_PUBLIC_IP>:/home/ubuntu/nsemargins/

Step 4 — Configure Environment Variables

cd /home/ubuntu/nsemargins

# Generate a secure secret key
python3 -c "import secrets; print(secrets.token_hex(32))"

# Create .env (never commit this file)
cat > .env << 'EOF'
SECRET_KEY=<paste-generated-key-here>
EOF
chmod 600 .env

Step 5 — Build and Start

cd /home/ubuntu/nsemargins

docker compose build        # ~2 minutes on first run
docker compose up -d        # start in background
docker compose logs -f      # watch startup

On first start the app downloads today's NSE data automatically. Watch for:

Bhavcopy: inserted 45066 contracts for 2026-04-10
SPAN XML: wrote 45066 RiskArray rows for 2026-04-10

The app is now available at http://<EC2_PUBLIC_IP>

Verify

  "data_mode": "span_file",
  "instrument_count": 45066,
